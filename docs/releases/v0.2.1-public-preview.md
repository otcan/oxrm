# v0.2.1-public-preview

First public preview release for oXRM.

## What Works

- Docker-first local runtime through `./oxrm`.
- Public-safe synthetic demo path:

```bash
./oxrm start
./oxrm ready
./oxrm seed job-search
./oxrm test
./oxrm urls
```

- Web UI dashboard, leads, tasks, events, settings, and saved views.
- HTTP APIs for health, views, XRM records, tasks, events, and outreach events.
- MCP tools/resources for queues, searches, saved views, records, events, and
  backup health.
- oXRM/`ork-linkedin` boundary documented as MCP/API-only.
- Production proxy Caddy config validates and preserves `/api/*` and `/mcp`
  semantics.
- Backup verification restores the latest dump into an isolated disposable
  database, checks schema readability, and drops the database before returning.

## Known Limitations

- Public demo data is synthetic only.
- No real LinkedIn automation is included in oXRM.
- Multi-tenant hosted SaaS, RBAC, and managed connectors are not part of this
  public preview.

## Upgrade

For local demo instances:

```bash
git pull
./oxrm reset
./oxrm start
./oxrm ready
./oxrm seed job-search
./oxrm test
```

For production-bound instances, configure backup credentials and use:

```bash
./oxrm upgrade
./oxrm test
```

Do not use `--skip-backup` for production-bound instances.

## Security And Privacy

- Do not commit real leads, real profile URLs, tokens, cookies, browser session
  data, logs, dumps, or backup artifacts.
- Keep `.env`, `instances/*.local.env`, and `.backups/` private.
- Report vulnerabilities privately through GitHub Security Advisories when
  available.
